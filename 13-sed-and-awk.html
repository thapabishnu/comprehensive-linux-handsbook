sed and awk

The grown-up text tools. 50 years old, on every Unix system, used in production every day.

§1Why this matters

When cut/sort/uniq aren't enough — when you need substitutions across files, line-range edits, fielded transforms, conditionals — these two cover it. You don't need to master them; you need literacy.

§2Mental model

  • sed = stream editor. Reads input line by line, applies editing commands to lines matching an address, prints the result.
  • awk = pattern-action language. For each line, finds fields split on whitespace (or your separator), then runs actions on lines matching patterns.
   sed:   [address]command/...      e.g.  s/old/new/g     1,5d
   awk:   pattern { action }        e.g.  $3 > 100 { print $1 }

§3sed essentials

Syntax shape

$ sed [-n] [-i] [-E] 'addr command' file
  • -n — suppress automatic printing (use with p).
  • -i — edit in place (modifies the file).
  • -E — Extended Regex.

Most-used commands

CommandEffect
s/old/new/Substitute first old per line
s/old/new/gSubstitute every occurrence per line
s/old/new/2Only the 2nd occurrence per line
s/old/new/ICase-insensitive substitution
dDelete matching lines
pPrint matching lines (use with -n)
qQuit immediately
NdDelete line N
1,5dDelete lines 1 through 5
/pat/dDelete lines matching pat
/start/,/end/pPrint from start match to end match

Examples

$ sed 's/error/ERROR/g' log.txt          # uppercase the word "error" everywhere
$ sed -E 's/[0-9]+/N/g' data.txt         # replace every number with literal N
$ sed -n '5,10p' file.txt                # print only lines 5-10
$ sed '/^#/d' /etc/ssh/ssh_config        # delete comment lines
$ sed -i.bak 's/old/new/g' file.txt      # in-place edit; backup to .bak
Mentor note · sed -i is permanent

Always pair with -i.bak (or dry-run first). Irreversibly mangling configs is a classic 2 AM mistake.

Multi-command sed

$ sed -e 's/foo/bar/g' -e 's/baz/quux/g' file.txt
$ sed 's/foo/bar/g; s/baz/quux/g' file.txt

§4awk essentials

Syntax shape

$ awk 'BEGIN { ... } pattern { action } END { ... }' file
  • BEGIN — runs once before any input.
  • pattern { action } — runs action for every line where pattern matches.
  • END — runs once after all input.

If you omit the pattern, the action runs on every line. If you omit the action, the default is { print }.

Built-in variables

VariableMeaning
$0The whole line
$1, $2, …Field 1, 2, … (whitespace by default)
NFNumber of fields on this line
$NFThe last field
NRNumber of records (lines) read so far
FSField separator (default: whitespace)
OFSOutput field separator
FILENAMECurrent input filename

Useful patterns

PatternMatches
/regex/Lines matching regex
$3 > 100Lines where field 3 > 100
NR == 1Just the first line
NR > 1Skip the header
$1 == "alice"Where field 1 equals "alice"
NR >= 5 && NR <= 10Lines 5–10

Examples

# Print the second field of every line
$ awk '{print $2}' file.txt

# Use comma as field separator
$ awk -F',' '{print $1, $3}' employees.csv

# Sum a numeric column
$ awk -F',' 'NR > 1 {sum += $4} END {print sum}' employees.csv

# Print lines where col 3 is "eng"
$ awk -F',' '$3 == "eng"' employees.csv

# Print last field of every line
$ awk '{print $NF}' /etc/passwd

# Format output with printf
$ awk -F',' 'NR>1 {printf "%-12s %8d\n", $2, $4}' employees.csv

The Hello World of awk

awk 'BEGIN { print "start"; total=0 }
     NR > 1 { total += $4 }
     END { print "total:", total }' \
  /path/to/data.csv

Pattern blocks, accumulator, summary at the end. Compare to Python — awk is terse.

§5Guided walkthrough

$ cd ~/linux-sandbox/text/

# --- sed ---
$ sed 's/Anderson/Andersen/g' names.txt
$ sed -n '5p' names.txt
$ sed '/^A/d' names.txt
$ sed -n '/Khan/,/Lopez/p' names.txt

# Edit a config-like file (dry run, then in place with backup)
$ cp lorem.txt lorem.bak
$ sed -E 's/^[a-z]/X/' lorem.bak             # preview
$ sed -E -i.bak 's/^[a-z]/X/' lorem.bak      # in place, backup .bak

# --- awk ---
$ awk -F',' '{print $1, $2}' employees.csv
$ awk -F',' 'NR>1 && $3=="eng" {print $2}' employees.csv
$ awk -F',' 'NR>1 {sum+=$4; n++} END {print "avg:", sum/n}' employees.csv

# Group salary totals by department
$ awk -F',' 'NR>1 {total[$3] += $4} END {
  for (d in total) printf "%-8s %d\n", d, total[d]
}' employees.csv | sort -k2 -rn

That last one is a one-line group-by-sum. In SQL: SELECT dept, SUM(salary) FROM employees GROUP BY dept.

§6Gotchas

Gotcha · sed -i is permanent

Pair with .bak until you trust it: sed -i.bak ....

Gotcha · macOS sed differs from GNU

On macOS, -i requires an argument (-i '' for no backup). On Linux, -i alone is fine.

Tip · quote your programs

Single quotes prevent the shell from eating special characters.

Mentor note · awk's default separator is whitespace

For CSV use -F','. For colon-delimited files like /etc/passwd, -F':'.

Mentor note · numeric vs string compare

awk's $2 > 100 does numeric automatically. But $2 > "100" is string compare. Be explicit if mixing: +$2 > 100.

§7On-the-spot exercises

E13.1Use sed to replace every Anderson with Andersen in names.txt.

Show answer
$ sed 's/Anderson/Andersen/g' ~/linux-sandbox/text/names.txt

E13.2Use sed to delete every comment line and blank line from /etc/ssh/ssh_config.

Show answer
$ sed -E '/^\s*(#|$)/d' /etc/ssh/ssh_config

Two patterns OR'd inside the address: starts with #, or empty line.

E13.3Use awk to print just the names and salaries from employees.csv.

Show answer
$ awk -F',' '{print $2, $4}' ~/linux-sandbox/text/employees.csv

E13.4Print only the engineers (department = "eng") with their salaries.

Show answer
$ awk -F',' '$3=="eng" {print $2, $4}' ~/linux-sandbox/text/employees.csv

You don't need an explicit if; the pattern $3=="eng" is the condition.

E13.5Compute the average salary across all employees.

Show answer
$ awk -F',' 'NR>1 {sum+=$4; n++} END {print sum/n}' ~/linux-sandbox/text/employees.csv

E13.6Show line numbers of every active config line in /etc/ssh/ssh_config.

Show answer
$ awk '!/^\s*(#|$)/ {print NR": "$0}' /etc/ssh/ssh_config

!/.../ is "doesn't match." Then print line number and the line.

E13.7Convert /etc/passwd into a CSV with header.

Show answer
$ {
  echo "username,uid,gid,home,shell"
  awk -F':' '{print $1","$3","$4","$6","$7}' /etc/passwd
} > /tmp/users.csv

The braces group echo and awk so their combined output is redirected.

E13.8From access.log, print all GET requests that returned 200, as "TIME PATH".

Show answer
$ awk '$6 ~ /"GET/ && $9 == 200 { print $4, $7 }' ~/linux-sandbox/text/access.log

~ is "matches regex." Common log format makes this trivial.

§8Real-world sysadmin scenario

Mass config update. A simple awk one: count nginx 5xx responses by endpoint:

$ awk '$9 ~ /^5/ { gsub(/"/, "", $7); count[$7]++ }
     END { for (p in count) print count[p], p }' \
   /var/log/nginx/access.log | sort -rn | head

A whole report in three lines. That's why awk survives — it's a programming language that fits in your head.

§9What to remember

  • sed for line-level edits (substitution, delete, print ranges).
  • awk for field-aware work (CSV, logs, structured text).
  • Quote your programs. Single quotes are friendlier.
  • sed -i.bak for safe in-place edits.
  • awk '$3 > 100 {print $1}' is the basic shape.
  • For group-by-sum analyses, awk's associative arrays shine.
