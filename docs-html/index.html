The Mentor's Guide
to Linux.

The labs ask you to prove you know Linux.
This guide teaches you Linux.

Who this is for

You. The student staring at a PDF that says "What does ls -ld /var/ do?" with no textbook, no slides, and a half-broken video.

I'll be your mentor. We go slow where it matters, fast where it doesn't, and every concept ends with drills you can run in your terminal right now and a real-world sysadmin scenario so you see how this stuff shows up on the job.

Finish the guide and the labs and you won't just pass NCS 205 — you'll have the foundation that senior Linux engineers operate on every day.

How to use this guide

  1. Set up your environment first — go to Orientation & setup. Pick WSL, a VM, or a cloud VPS.
  2. Work the modules in order. They build on each other.
  3. Do the drills as you read. Try them before expanding the answer.
  4. Read the sysadmin scenario at the end. That's the bridge from "memorized" to "I can fix things."
  5. Then attempt the matching lab PDFs. Each module says which lab numbers it prepares you for.

A note on philosophy

There is no shortcut to being good at Linux. There is only type, read, think, repeat. I'll make the path as efficient as I can, but the keyboard part is on you.

If a command doesn't make sense, run it. If it still doesn't make sense, break it on purpose and see what happens. Linux is one of the few systems where you learn fastest by deliberately messing things up in a safe sandbox.
