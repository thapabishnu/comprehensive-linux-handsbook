MODULE16 TIME~30 min

Users, groups, and sudo

Every audit, every breach investigation, every "why can't I access this?" ticket starts here.

В§1Why this matters

Multi-user is Linux's deep DNA. The privilege model вҖ” who can do what вҖ” is what keeps systems safe.

В§2Mental model

A user has a numeric UID. UID 0 is root вҖ” unrestricted. System accounts have low UIDs (1вҖ“999); real humans usually start at UID 1000.

A group has a numeric GID. Every user has one primary group and zero or more supplementary groups.

The kernel checks access against UID + groups + permission bits. Everything else (sudo, PAM, ACLs) sits on top of this core model.

The four key files

FileContains
/etc/passwdAccount info вҖ” username, UID, GID, home, shell. World-readable.
/etc/shadowHashed passwords + aging info. Root-readable only.
/etc/groupGroup definitions and members.
/etc/sudoers + /etc/sudoers.d/Who can sudo. Edit with visudo, never directly.

В§3Core commands

Inspecting accounts

CommandWhat it shows
whoamiYour username
idYour UID, GID, supplementary groups
id aliceSame for another user
groupsGroups you're in
getent passwd aliceLike grepping /etc/passwd but also queries LDAP/AD if configured
last aliceRecent logins by alice
lastlogLast login of every user

Creating, modifying, deleting

CommandEffect
sudo useradd -m -s /bin/bash -G sudo aliceCreate user alice with home dir, bash shell, in group sudo
sudo passwd aliceSet/reset alice's password
sudo usermod -aG devs aliceAppend to supplementary group (always use -a!)
sudo usermod -L aliceLock the account
sudo usermod -U aliceUnlock
sudo userdel -r aliceDelete user and home dir
sudo groupadd devsCreate a group
chage -l aliceShow password aging info

Privilege escalation

CommandEffect
sudo cmdRun cmd as root (if in sudoers)
sudo -u alice cmdRun cmd as alice
sudo -iStart interactive login root shell
sudo -sRoot shell, keep your env
sudo -lList what sudo rights you have
su -Switch to root (requires root password)
visudoSafely edit /etc/sudoers
visudo -f /etc/sudoers.d/myruleEdit a drop-in fragment

В§4Guided walkthrough

$ whoami
$ id
$ groups
$ grep "$USER" /etc/passwd

# create a test user
$ sudo useradd -m -s /bin/bash -c "Test User" testdev
$ sudo passwd testdev

$ id testdev
$ sudo getent passwd testdev
$ sudo ls -ld /home/testdev

# groups
$ sudo groupadd devs
$ sudo usermod -aG devs testdev
$ id testdev
$ groups testdev

$ sudo su - testdev
$ exit

# cleanup
$ sudo userdel -r testdev
$ sudo groupdel devs
Mentor note В· -aG vs -G

usermod -G devs alice replaces alice's supplementary groups with just devs. Use -aG (append) to add without removing. Stripping a teammate out of sudo by forgetting -a is a once-and-never-again lesson.

sudo configuration

To allow alice to run anything as root without a password (dev VM only вҖ” never prod):

$ sudo visudo -f /etc/sudoers.d/alice
alice ALL=(ALL) NOPASSWD: ALL

For finer control вҖ” alice can only restart nginx:

alice ALL=(root) NOPASSWD: /bin/systemctl restart nginx

Never edit /etc/sudoers directly. A syntax error can lock you out entirely.

В§5Gotchas

Mentor note В· /etc/passwd is world-readable

Many tools need to map UIDs to names. But it means anything in the GECOS field is public.

Gotcha В· -aG vs -G

Reread above. This bug costs careers.

Gotcha В· removing a user doesn't clean up everywhere

userdel -r removes home dir; files in /tmp or /srv you must hunt down.

Mentor note В· sudo keeps its own environment

sudo CMD doesn't see your aliases or functions; sudo -E CMD preserves env. PATH is filtered for security.

Mentor note В· su vs sudo -i

su - requires root's password. sudo -i uses your password (via sudoers). Most modern systems disable direct root login.

Gotcha В· locked вү  disabled

passwd -l alice locks the password but SSH keys still work. To fully prevent login, also set shell to /sbin/nologin and remove SSH keys.

В§6On-the-spot exercises

E16.1What's your UID and primary GID?

Show answer
$ id -u
$ id -g

E16.2List every account on the system whose shell is /bin/bash.

Show answer
$ grep '/bin/bash$' /etc/passwd | cut -d':' -f1

Roughly your set of human users (system accounts usually have /usr/sbin/nologin).

E16.3Add yourself to group docker.

Show answer
$ sudo usermod -aG docker "$USER"
# log out and back in for the new group to take effect

E16.4Show how many days until your password expires.

Show answer
$ chage -l "$USER"

E16.5Create a service account backup-svc that has no shell login.

Show answer
$ sudo useradd -r -s /usr/sbin/nologin -m -d /var/lib/backup-svc backup-svc

-r = system account; -s /usr/sbin/nologin = no interactive login.

E16.6List exactly what sudo rights you have.

Show answer
$ sudo -l

E16.7Switch to user nobody and run a command.

Show answer
$ sudo -u nobody whoami

Runs one command and returns.

E16.8Configure alice to restart nginx via sudo without a password, but nothing else.

Show answer
$ sudo visudo -f /etc/sudoers.d/alice-nginx
alice ALL=(root) NOPASSWD: /bin/systemctl restart nginx, /bin/systemctl reload nginx

В§7Real-world sysadmin scenario

New engineer joining.

$ sudo useradd -m -s /bin/bash -c "Jamie Lee" jamie
$ sudo passwd jamie

$ sudo usermod -aG devs,docker,sudo jamie

$ sudo install -d -m 700 -o jamie -g jamie /home/jamie/.ssh
$ printf '%s\n' "ssh-ed25519 AAAA... jamie@laptop" | \
  sudo install -m 600 -o jamie -g jamie /dev/stdin /home/jamie/.ssh/authorized_keys

$ id jamie
$ sudo -u jamie -i whoami

When jamie leaves:

$ sudo passwd -l jamie                    # lock immediately
$ sudo usermod -e $(date +%F) jamie       # expire account today
$ sudo tar -cJf /backup/jamie-home.tar.xz /home/jamie
$ sudo userdel -r jamie

Lock first (immediate), then back up, then delete. Order matters.

В§8What to remember

  • UID/GID + permission bits = whole privilege model.
  • id, groups, getent вҖ” your inspectors.
  • useradd -m -s /bin/bash NAME is the standard create.
  • usermod -aG when adding to groups. Don't forget -a.
  • visudo only; never edit /etc/sudoers directly.
  • sudo -l shows what you can do.
  • Lock-then-delete when offboarding.
