The vi / vim editor

Every Linux box has vi. Not "almost every." Every. Five minutes of learning gets you "save and quit"; two hours gets you faster than any GUI editor.

§1Why this matters

You will SSH into a stripped-down container or a rescue boot environment where nano doesn't exist, and vi will be your only editor. Even when you have nicer options, vi's modal model is so efficient that millions of professionals keep using it daily.

§2Mental model

vi is modal. Every keypress means different things depending on which mode you're in:

                ┌──────────────┐
       Esc      │   NORMAL     │  i / a / o / I / A / O
   ┌──────────► │  (default)   ├────────────────────────────┐
   │            │ — moves &    │                            │
   │            │   commands   │                            ▼
   │            └──────┬───────┘                    ┌──────────────┐
   │                   │                            │   INSERT     │
   │                   │ :                          │  (typing)    │
   │                   ▼                            └──────────────┘
   │            ┌──────────────┐
   │            │  COMMAND-LINE│      Enter
   │            │      :       │
   └──────────  │ (save, quit, │
                │  search)     │
                └──────────────┘
  • Normal mode (default) — keys are commands. j moves down, dd deletes a line.
  • Insert mode — keys insert text. i to enter, Esc to leave.
  • Command-line mode — : followed by an ex command. :w save, :q quit, :s/foo/bar/g substitute.

The reason vi feels alien is that you're not "typing into a document." You're issuing commands to a tiny programming language whose only data is the buffer.

§3The survival kit

If you remember nothing else, remember these:

To do thisPress
Open a filevi filename
Enter insert modei
Leave insert modeEsc
Save:w Enter
Quit:q Enter
Save & quit:wq Enter
Quit without saving:q! Enter
Undou
RedoCtrl+R

That's enough to never be stranded.

§4Commands by category

Movement (Normal mode)

KeysMove
h j k lLeft, down, up, right (arrows work, but break the flow)
w / bForward / back one word
0 / ^Beginning of line (0 = column 0; ^ = first non-blank)
$End of line
gg / GTop / bottom of file
:NJump to line N (e.g. :42)
Ctrl+f / Ctrl+bPage down / up
%Jump to matching bracket/paren
*Search forward for word under cursor

Editing (Normal mode)

KeysEffect
i / aInsert before / after cursor
I / AInsert at beginning / end of line
o / OOpen new line below / above
xDelete character under cursor
ddDelete current line (and yank it)
dwDelete from cursor to end of word
d$ / DDelete to end of line
yyYank (copy) current line
p / PPaste after / before cursor
r<char>Replace one character
cwChange word (delete + enter insert)
ccChange whole line
u / Ctrl+RUndo / redo
.Repeat last edit — incredibly powerful

Search & replace (Command-line mode)

CommandEffect
/fooSearch forward. n next, N previous.
?fooSearch backward.
:%s/foo/bar/gReplace every foo with bar in the file.
:%s/foo/bar/gcSame, but confirm each replacement.
:5,20s/foo/bar/gOnly between lines 5 and 20.
:nohClear search highlight.

File / window commands

CommandEffect
:wWrite (save).
:w newnameSave as.
:qQuit. Errors if buffer is modified.
:q!Quit, discard changes.
:wq or ZZWrite and quit.
:e other.txtOpen another file in this window.
:sp / :vspSplit horizontally / vertically.
Ctrl+W h/j/k/lMove between splits.

The grammar

Most edit commands follow [count] operator motion:

  • 5dd — delete 5 lines.
  • d3w — delete 3 words.
  • y$ — yank to end of line.
  • 2cc — change 2 lines.

Memorize the operators (d, y, c) and the motions (w, $, gg), and you can compose dozens of commands.

§5Guided walkthrough

Open a scratch file:

$ vi ~/vi-practice.txt

You're in Normal mode. Type these in order:

i                   ← enter insert mode
The quick brown fox
jumps over the lazy dog
[Esc]               ← back to Normal
gg                  ← top of file
yy                  ← yank (copy) the first line
p                   ← paste below — now there are 3 lines
:                   ← command-line mode
%s/fox/cat/g        ← every "fox" becomes "cat"
[Enter]
u                   ← undo (cat → fox)
Ctrl+R              ← redo
:w                  ← save
:q                  ← quit

You just edited a file using nothing but vi. That's the survival kit plus a substitution, an undo, and a yank-paste.

A more practical drill

Edit /etc/hosts (read-only first — we'll undo any change):

$ sudo vi /etc/hosts

Inside:

  • gg — top of file
  • /127 — find the first 127.
  • n — next match
  • :q! — quit without saving

§6Gotchas

Gotcha · Caps-Lock is your enemy

J joins lines; K opens the man page. Many of vi's "weird" behaviors are you accidentally in caps.

Tip · when in doubt, Esc

Press Esc. Twice if you're really lost.

Gotcha · :q refuses on unsaved changes

Use :q! to discard, or :wq to save first.

Gotcha · Ctrl+S freezes your terminal

XOFF. Press Ctrl+Q to unfreeze. (Old TTY thing — vi inherited it.)

Mentor note · . repeats the last edit

Not the last motion. After dw, pressing . deletes another word. The day . clicks for you is the day vi becomes fun.

Gotcha · :set paste before pasting

Otherwise vim's auto-indent rewrites your text into a mess. :set nopaste after.

§7On-the-spot exercises

E6.1Open ~/vi-practice.txt (create it if needed), type three lines of text, save, and quit.

Show answer
vi ~/vi-practice.txt        ← opens
i                           ← insert mode
line one
line two
line three
[Esc]
:wq                         ← write + quit

E6.2Re-open the file. Move to line 2. Delete it. Save.

Show answer
vi ~/vi-practice.txt
:2          ← jump to line 2
dd          ← delete the line
:w          ← save

E6.3In the same file, replace every occurrence of "line" with "row".

Show answer
:%s/line/row/g

% = "all lines in the file." g = "every occurrence on each line" (without g, only the first per line).

E6.4Undo the substitution.

Show answer
u

Press Ctrl+R to redo.

E6.5Yank the first line, paste it three times at the end of the file.

Show answer
gg          ← top
yy          ← yank line 1
G           ← bottom
p           ← paste below current line
p
p

E6.6Search for "row" and jump through every match. Quit without saving.

Show answer
/row[Enter]
n           ← next match
n           ← next
:q!         ← quit, discard

E6.7Open /etc/hosts in read-only mode.

Show answer
$ view /etc/hosts

Equivalent to vi -R /etc/hosts. The :q works as normal; the file is marked read-only.

E6.8In ~/vi-practice.txt, indent every line by 4 spaces. (Hint: visual mode V + >.)

Show answer

Several routes. Simplest with visual line mode:

gg          ← top
V           ← visual line mode
G           ← extend to bottom
>           ← shift right (indent) one shiftwidth
:set shiftwidth=4   ← if not already 4

Or with a substitution:

:%s/^/    /

§8Real-world sysadmin scenario

Production push goes wrong. A coworker SSHs you screen-share: "Can you change the timeout in this nginx config?" You have one vi window, a flaky connection, and no time to learn.

You do this:

vi /etc/nginx/conf.d/api.conf      ← open
/proxy_read_timeout                ← search for the line
n                                  ← skip to the right occurrence
$                                  ← end of line
cw                                 ← change word (the value)
60s[Esc]                           ← type new value, leave insert
:w                                 ← save
:q                                 ← quit

Then back in shell:

$ sudo nginx -t          # validate syntax
$ sudo systemctl reload nginx

If syntax check fails, you go right back to vi, fix the typo, save, validate, reload. The whole loop runs entirely in your terminal. No mouse, no window switching. This is the loop you'll run hundreds of times in your career. Make vi cheap.

§9What to remember

  • Vi is modal. Normal = commands, Insert = typing, Command-line = :.
  • Survival kit: i, Esc, :w, :q, :wq, :q!, u, Ctrl+R.
  • Editing grammar: [count] operator motion.
  • . repeats the last edit — the secret weapon.
  • view opens read-only. Use it when you only mean to look.
