Shell scripting

A shell script is a one-liner that kept its job. 500 lines of Python will be replaced; a tight 30-line bash script lives for a decade.

§1Why this matters

Every recurring task — daily backups, log rollups, deploy sequences, the cron job that emails the team a Friday report — is someone's shell script. Writing a clean, defensive, portable script is one of the highest-leverage skills in this whole guide.

§2Mental model

A shell script is just a file the shell reads top to bottom. Same syntax as the prompt. The shebang picks the interpreter:

#!/bin/bash
# this is bash
set -euo pipefail
echo "hello"
  • #!/bin/bash — run with bash.
  • #!/bin/sh — run with POSIX shell (more portable).
  • set -e — exit on the first error.
  • set -u — error on unset variables.
  • set -o pipefail — a pipeline fails if any stage fails.

set -euo pipefail is the single biggest improvement most scripts could get. It turns silent failures into loud ones.

§3Core building blocks

Variables and quoting

$ name="Sarah"
$ echo "Hello, $name"            # "Hello, Sarah"
$ echo 'Hello, $name'            # 'Hello, $name'   ← single quotes don't expand
$ echo "Hello, ${name}!"         # braces disambiguate

# Command substitution
$ today=$(date +%F)
$ echo "Today is $today"

# Math (bash)
$ count=$((3 + 4))

Quote your variables. "$file" not $file. Otherwise a filename with a space becomes two arguments.

Conditionals

if [[ -f /etc/hosts ]]; then
  echo "exists"
elif [[ -d /etc ]]; then
  echo "/etc is a dir"
else
  echo "neither"
fi

# String comparison
[[ "$name" == "Sarah" ]] && echo "hi Sarah"

# Numeric comparison
(( count > 5 )) && echo "big"

[[ ... ]] is bash's improved test. Use it over the old [ ... ] when you can.

Common file tests:

TestTrue if
-e fexists
-f fregular file
-d fdirectory
-r freadable
-w fwritable
-x fexecutable
-s fnon-empty
f1 -nt f2f1 newer than f2

Loops

# for over a list
for h in web-01 web-02 web-03; do
  echo "$h"
done

# for over glob (carefully)
for log in /var/log/*.log; do
  [[ -f "$log" ]] || continue
  echo "$log"
done

# while reading lines
while IFS= read -r line; do
  echo "got: $line"
done < /etc/hostname

# numeric C-style
for ((i=1; i<=5; i++)); do echo $i; done

Arguments and exit codes

#!/bin/bash
set -euo pipefail

# Positional args
echo "script: $0"
echo "first arg: $1"
echo "all args: $@"
echo "count: $#"

# Defaults
: "${TIMEOUT:=30}"
echo "timeout is $TIMEOUT"

# Functions
greet() {
  local who="$1"
  echo "hi, $who"
}
greet "world"

# Exit codes
exit 0      # success
exit 1      # general failure

Useful idioms

# default if unset
: "${LOG:=/tmp/script.log}"

# log to both file and stdout
exec > >(tee -a "$LOG") 2>&1

# trap cleanup on exit
cleanup() {
  rm -f /tmp/myscript.$$.tmp
}
trap cleanup EXIT

# require root
[[ $EUID -eq 0 ]] || { echo "must run as root" >&2; exit 1; }

# require a command exists
command -v jq >/dev/null 2>&1 || { echo "needs jq"; exit 1; }

§4Guided walkthrough

Make a real script — ~/bin/dailyreport.sh:

$ mkdir -p ~/bin
$ cat > ~/bin/dailyreport.sh <<'EOF'
#!/bin/bash
set -euo pipefail

# === defaults ===
: "${LOG_DIR:=/var/log}"
: "${OUT_FILE:=$HOME/daily-$(date +%F).txt}"

# === main ===
{
  echo "Daily report — $(date)"
  echo "host: $(hostname)"
  echo
  echo "## disk usage"
  df -h | grep -E '^/dev|Filesystem'
  echo
  echo "## top 5 largest files in $LOG_DIR"
  sudo find "$LOG_DIR" -type f -size +1M -printf '%s %p\n' 2>/dev/null \
    | sort -rn | head -5 \
    | awk '{ printf "  %6.1fM  %s\n", $1/1024/1024, $2 }'
  echo
  echo "## recent failed logins"
  sudo lastb -n 5 2>/dev/null || echo "  (lastb unavailable)"
} > "$OUT_FILE"

echo "wrote $OUT_FILE"
EOF

$ chmod +x ~/bin/dailyreport.sh
$ ~/bin/dailyreport.sh
$ cat ~/daily-*.txt

Read every line. { ... } > file is command grouping with redirection — every command inside the braces writes to the same file.

§5Gotchas

Gotcha · word splitting on unquoted variables

cp $file /tmp/ breaks if $file has a space. Always cp "$file" /tmp/.

Mentor note · == vs =

Inside [[ ... ]], both work for equality. Inside [ ... ] (legacy), only = is portable.

Gotcha · set -e doesn't catch errors inside if/||/&&

This is by design (so you can test commands), but surprises beginners.

Gotcha · pipes mask errors

cmd1 | cmd2 exit code is cmd2's. Use set -o pipefail and check ${PIPESTATUS[@]}.

Tip · bash -n script.sh

Syntax check without running. Use it before deploying.

Tip · install shellcheck

The script-linter you should install. It catches 90% of bash bugs before you ship.

Mentor habit · every non-trivial script

Starts with #!/bin/bash, set -euo pipefail, a usage function, and a top-of-file comment. Ten extra seconds; pays for itself the first time you come back six months later.

§6On-the-spot exercises

E14.1Write ~/bin/hello.sh that prints "hello from <hostname>" and exits 0. Make it executable.

Show answer
$ cat > ~/bin/hello.sh <<'EOF'
#!/bin/bash
echo "hello from $(hostname)"
EOF
$ chmod +x ~/bin/hello.sh
$ ~/bin/hello.sh

E14.2Extend it: if $1 is passed, greet that name; otherwise "world."

Show answer
#!/bin/bash
who="${1:-world}"
echo "hello, $who, from $(hostname)"

${1:-world} = "use $1 if set, otherwise 'world'."

E14.3Script that takes a directory and prints how many .log files it contains. Error if dir doesn't exist.

Show answer
#!/bin/bash
set -euo pipefail

dir="${1:-}"
[[ -d "$dir" ]] || { echo "usage: $0 <dir>" >&2; exit 1; }

count=$(find "$dir" -maxdepth 1 -name '*.log' -type f | wc -l)
echo "$count .log files in $dir"

E14.4One-liner: "OK" if /etc/hostname exists and is readable, else "MISSING".

Show answer
$ [[ -r /etc/hostname ]] && echo OK || echo MISSING

E14.5Loop over every user in /etc/passwd and print "username -> home".

Show answer
while IFS=':' read -r user _ _ _ _ home _; do
  echo "$user -> $home"
done < /etc/passwd

IFS=':' lets the line split on colons. _ = "I don't care about this field."

E14.6Backup helper: copy a file to <file>.bak.YYYY-MM-DD. Refuse if source doesn't exist.

Show answer
#!/bin/bash
set -euo pipefail

src="${1:-}"
[[ -f "$src" ]] || { echo "usage: $0 <file>" >&2; exit 1; }

dst="${src}.bak.$(date +%F)"
cp -p "$src" "$dst"
echo "backup: $dst"

E14.7Function confirm that prompts "Continue? [y/N]" and returns non-zero unless 'y'.

Show answer
confirm() {
  read -r -p "Continue? [y/N] " ans
  [[ "$ans" =~ ^[yY]$ ]]
}

confirm && echo "going!" || { echo "abort"; exit 1; }

=~ is bash's regex match operator inside [[ ... ]].

E14.8Run shellcheck on your dailyreport.sh.

Show answer
$ sudo apt install -y shellcheck
$ shellcheck ~/bin/dailyreport.sh

shellcheck will point out quoting issues, unset variables, command substitution warnings — listen to it.

§7Real-world sysadmin scenario

Preflight script before deploy. Must: (1) confirm we're on the deploy box, (2) clean git tree, (3) tests pass.

#!/bin/bash
# preflight.sh — gate before deploy
set -euo pipefail

EXPECTED_HOST="deploy-01.prod"

# 1. host check
[[ "$(hostname)" == "$EXPECTED_HOST" ]] || {
  echo "ERROR: must run on $EXPECTED_HOST, you're on $(hostname)" >&2
  exit 1
}

# 2. clean working tree
cd /srv/app
if ! git diff --quiet || ! git diff --cached --quiet; then
  echo "ERROR: working tree is dirty" >&2
  git status --short
  exit 2
fi

# 3. tests
if ! ./run-tests.sh; then
  echo "ERROR: tests failed" >&2
  exit 3
fi

echo "OK — preflight passed at $(date -Iseconds)"

25 lines that has saved real deployments. Note the distinct exit codes — your wrapper can decide what to do based on which check failed.

§8What to remember

  • Top of every script: #!/bin/bash and set -euo pipefail.
  • Quote your variables. Always.
  • [[ ... ]] for tests; (( ... )) for math.
  • ${VAR:-default}, ${VAR:?error} — learn parameter expansion gradually; it's powerful.
  • trap cleanup EXIT for cleanup on exit.
  • Run shellcheck on every script.
  • Define exit codes deliberately — they're your script's API.
